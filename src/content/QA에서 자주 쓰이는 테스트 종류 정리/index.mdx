몽키테스팅:

- 임의의 입력값으로 동작을 확인
- 예상치 못한 에러를 찾아냄
- 개발자의 의도를 고려하지 않고 즉흥적으로 조작함
- 버그를 재현하기 어렵거나 불가능한 경우가 있음

Ad-hoc 테스팅:

- 테스터의 경험과 직관에 의존해서 즉흥적으로 테스팅함
- 정식 ㅌ스트로 발견하기 어려운 결함을 발견할 수 있음

탐색적 테스팅:

- 도메인 지식과 경험을 활용해서 테스트를 설계하고 실행
- 테스터의 경험에 기반해 테스트 설계와 실행이 동시에 이뤄짐
- 시간 제한을 두고 테스트 차터를 수행

스모크 테스팅:

- 핵심 기능이 제대로 작동하는지 빠르게 확인하는 테스트
- 새로운 빌드가 출시 되기 전, 핵심 기능이 제대로 작동하는지 확인

확인 테스팅:

- 결함이 성공적으로 제거되었는지 확인하는 테스트

리그레이션 테스팅:

- 변경 이후에 수정되지 않았거나 발견되지 않은 결함을 발견하는 테스트
